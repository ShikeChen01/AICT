# Self-Hosted PostgreSQL VM

PostgreSQL 16 running in Docker on a GCE VM, connected from Cloud Run via a Serverless VPC Connector.

## Architecture

```
Cloud Run (us-central1)
    │
    ├── VPC Connector (aict-vpc-connector)
    │       │
    │       ▼
    │   VM: aict-postgres (10.128.0.5:5432)
    │       └── Docker: postgres:16-alpine (SSL enabled)
    │
    └── HTTP :9090
        VM: aict-sandbox (sandbox pool manager)
```

## Initial Setup

1. SSH into the VM:
   ```bash
   gcloud compute ssh aict-postgres --zone=us-central1-f --project=aict-487016
   ```

2. Clone the repo (or scp the `infra/postgres/` directory):
   ```bash
   git clone <repo-url> /tmp/aict
   ```

3. Run the setup script as root:
   ```bash
   sudo /tmp/aict/infra/postgres/scripts/setup.sh
   ```

4. The script will output the generated password and the env vars to add to `.env.development`.

## Directory Layout on the VM

```
/opt/postgres/
├── compose/
│   ├── docker-compose.yml
│   ├── postgresql.conf
│   ├── pg_hba.conf
│   └── .env               # credentials (generated by setup.sh)
├── data/                   # pgdata volume (boot disk or /mnt/pgdata)
├── ssl/
│   ├── server.crt          # self-signed, 10-year validity
│   └── server.key
└── backups/                # pg_dump output
```

If a data disk is attached at `/dev/sdb`, the setup script will format it, mount it at `/mnt/pgdata`, and store data + backups there instead.

## Operations

### Check status
```bash
docker ps                                    # container running?
docker exec aict-postgres pg_isready -U aict # accepting connections?
docker logs aict-postgres --tail 50          # recent logs
```

### Restart
```bash
cd /opt/postgres/compose && docker compose restart
```

### Manual backup
```bash
sudo /opt/postgres/compose/scripts/backup.sh
ls -la /opt/postgres/backups/
```

### Restore from backup
```bash
sudo /opt/postgres/compose/scripts/restore.sh /opt/postgres/backups/aict_YYYYMMDD_HHMMSS.sql.gz
```

### Automated daily backup (cron)
```bash
sudo crontab -e
# Add:
0 3 * * * /opt/postgres/compose/scripts/backup.sh >> /var/log/pg-backup.log 2>&1
```

### Resize boot disk (online, no downtime)
```bash
# From your local machine:
gcloud compute disks resize aict-postgres --size=256GB --zone=us-central1-f --project=aict-487016

# Then SSH into the VM:
sudo resize2fs /dev/sda1
```

## Networking

- **VPC Connector**: `aict-vpc-connector` in `us-central1`, subnet `10.8.0.0/28`
- **Firewall rule**: `allow-postgres-from-vpc-connector` allows TCP 5432 from `10.8.0.0/28` to VMs tagged `postgres-vm`
- **SSL**: Self-signed cert; `DB_SSL_MODE=require` in Cloud Run env vars

## Environment Variables

These go in `.env.development` on your local machine:

| Variable | Description |
|---|---|
| `POSTGRES_VM_HOST` | VM internal IP (`10.128.0.5`) |
| `POSTGRES_VM_PORT` | `5432` |
| `POSTGRES_VM_USER` | `aict` |
| `POSTGRES_VM_PASSWORD` | Generated by setup.sh |
| `POSTGRES_VM_DB` | `aict` |
| `DB_SSL_MODE` | `require` (VM) or `disable` (local dev) |
| `VPC_CONNECTOR_NAME` | `aict-vpc-connector` |

## Upgrading PostgreSQL

1. Backup: `sudo /opt/postgres/compose/scripts/backup.sh`
2. Update `docker-compose.yml` image tag (e.g., `postgres:17-alpine`)
3. `cd /opt/postgres/compose && docker compose pull && docker compose up -d`
4. Verify: `docker exec aict-postgres psql -U aict -c "SELECT version();"`

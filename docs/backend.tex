\section{Error Handling Strategy}
%% ============================================================

\begin{verbatim}
+------------------------------------------------------------------+
|                    ERROR HANDLING LAYERS                          |
+------------------------------------------------------------------+
|                                                                   |
|   Layer 1: API Validation (Pydantic)                              |
|   +------------------------------------------------------------+  |
|   | Invalid request body -> 422 Unprocessable Entity           |  |
|   | Missing required field -> 422 with field details           |  |
|   +------------------------------------------------------------+  |
|                                                                   |
|   Layer 2: Authentication                                         |
|   +------------------------------------------------------------+  |
|   | Missing token -> 401 Unauthorized                          |  |
|   | Invalid token -> 401 Unauthorized                          |  |
|   +------------------------------------------------------------+  |
|                                                                   |
|   Layer 3: Business Logic                                         |
|   +------------------------------------------------------------+  |
|   | Task not found -> 404 Not Found                            |  |
|   | Agent not found -> 404 Not Found                           |  |
|   | Scope violation -> 403 Forbidden                           |  |
|   | Git blocked cmd -> 403 Forbidden                           |  |
|   +------------------------------------------------------------+  |
|                                                                   |
|   Layer 4: External Services                                      |
|   +------------------------------------------------------------+  |
|   | E2B failure -> 503 Service Unavailable + retry             |  |
|   | LLM timeout -> 504 Gateway Timeout + retry                 |  |
|   | Git failure -> 500 + error details                         |  |
|   +------------------------------------------------------------+  |
|                                                                   |
|   All errors logged with:                                         |
|   - Timestamp                                                     |
|   - Request ID                                                    |
|   - Agent ID (if applicable)                                      |
|   - Stack trace                                                   |
|                                                                   |
+------------------------------------------------------------------+
\end{verbatim}

# How to Apply the Migration Restructure

## What Changed

The 29 hand-written incremental migrations have been replaced with a single
baseline migration derived from `models.py`. Going forward, all migrations
are autogenerated from model diffs.

## Files to Replace

```
# DELETE these (old migration chain):
backend/migrations/versions/001_init_mvp0_schema.py  through  029_mcp_server_configs.py
backend/scripts/safe_migrate.py
backend/tests/test_safe_migrate.py

# REPLACE these:
backend/alembic.ini               ← migration_restructure/alembic.ini
backend/migrations/env.py          ← migration_restructure/migrations/env.py
backend/db/migration_runner.py     ← migration_restructure/db/migration_runner.py
backend/tests/test_migration_graph.py  ← migration_restructure/tests/test_migration_graph.py

# ADD these (new files):
backend/migrations/versions/001_baseline.py  ← migration_restructure/migrations/versions/001_baseline.py
backend/scripts/lint_migrations.py           ← migration_restructure/scripts/lint_migrations.py
backend/scripts/new_migration.sh             ← migration_restructure/scripts/new_migration.sh
backend/MIGRATIONS.md                        ← migration_restructure/MIGRATIONS.md
```

## Steps

1. Remove the 29 old migration files from `backend/migrations/versions/`
2. Copy the new files into place (see mapping above)
3. Delete `backend/scripts/safe_migrate.py` and `backend/tests/test_safe_migrate.py`
4. Drop your dev database and recreate:
   ```bash
   dropdb aict && createdb aict
   alembic -c backend/alembic.ini upgrade head
   ```
5. Verify autogenerate sees no diff:
   ```bash
   alembic -c backend/alembic.ini check
   ```

## Creating Your Next Migration

```bash
# 1. Edit backend/db/models.py
# 2. Generate:
./backend/scripts/new_migration.sh "add user avatar column"
# 3. Review, test, commit.
```
